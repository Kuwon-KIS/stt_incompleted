# 배치 날짜 기본값 마지막 날짜 전환 계획

작성일: 2026-03-20
범위: 배치 처리 탭 달력 초기 선택값을 시작일(min)에서 마지막 날짜(max)로 변경

## 목적
- 최근 데이터 중심 업무 흐름에 맞게 초기 선택을 최신 날짜로 맞춘다.
- 기존 날짜 범위 선택 UX를 유지하면서 초기 편의성을 개선한다.

## 구현 포인트
- 프론트 달력 초기화 로직의 defaultDate 기준을 max_date로 변경한다.
- no data, single date, 모바일 입력 동작은 기존과 동일하게 유지한다.

## 대상 파일
- app/static/js/app.js
- (필요 시) app/static/index.html

## 상세 작업
1. initializeCalendar의 defaultDate를 minDate에서 maxDate로 전환
2. 초기 렌더 직후 날짜 입력 필드 동기화 동작 점검
3. 자동 분석 트리거가 의도치 않게 즉시 호출되지 않는지 확인

## 검증
- 배치 탭 최초 진입 시 마지막 날짜가 기본 선택되는지 확인
- 단일 날짜만 존재할 때 정상 선택되는지 확인
- available dates가 비어 있을 때 예외 없이 렌더되는지 확인
- 브라우저/모바일에서 선택 동작 회귀 없는지 확인

## 완료 기준
- 초기 선택이 max_date로 일관되게 적용
- 기존 범위 선택 동작 및 API 호출 흐름 회귀 없음
